# 1. プロジェクト目的

Windows向けの純粋なネイティブテキストエディタを開発する。

本ソフトウェアは以下のエディタの長所を取り入れ、それらを超えることを目的とする。

- 秀丸エディタ
- サクラエディタ
- MIFES

本プロジェクトではUIやソースコードを模倣するのではなく、思想・操作性・使い勝手を参考にした独自実装を行う。

本ソフトは「Windows最速・最軽量」を最優先とし、AI時代の新しいスタンダードエディタを目指す。

---

# 2. 基本コンセプト

開発方針は以下を絶対条件とする。

- 起動が圧倒的に速い
- 動作が極めて軽い
- メモリ消費を最小限にする
- 巨大ファイルでも快適に動作する
- 日本語編集に強い
- プログラマ向け機能が充実
- AIとの親和性を重視
- カスタマイズ性を極限まで高める
- Windowsネイティブアプリとして実装する

Electronは禁止。

.NET MAUIは禁止。

WebViewは禁止。

Chromiumは禁止。

# 19. 開発ルール（Claude Codeへの指示）

あなたは本プロジェクトのテックリード兼シニアソフトウェアアーキテクトとして振る舞うこと。

以下を必ず守ること。

- 常に長期保守性を最優先する。
- 可読性より性能が重要となる箇所は、その理由をコメントで明示した上で最適化を行う。
- 推測で実装せず、不明点はIssueとして整理する。
- 巨大なクラスや関数を作らず、責務を明確に分離する。
- 機能追加時に既存コードを破壊しない設計を維持する。
- 外部ライブラリへの依存は最小限とし、採用理由を文書化する。
- すべての機能に対して設計書・テスト・実装をセットで進める。
- Pull Request単位でレビュー可能な粒度に分割して実装する。
- 常にリファクタリング候補を提示するが、ユーザーの承認なしに大規模変更は行わない。
- ベンチマークを重視し、性能改善は計測結果を根拠として行う。
- 各フェーズ終了時に設計・実装・テスト・残課題をレポートする。

# 20. 最終目標

本プロジェクトの最終目標は、

「秀丸エディタの軽さ」
「MIFESの操作性」
「サクラエディタの拡張性」

これらを融合し、さらに

- AI統合
- 超高速ログ解析
- 巨大ファイル編集
- Git統合
- LSP統合
- 高度な検索性能

を備えた、Windowsにおける次世代の定番ネイティブテキストエディタを実現することである。

## 成功基準（Definition of Done）

- 起動速度・メモリ消費・編集性能が主要既存エディタと比較して競争力を持つこと
- 主要機能に対する自動テストが整備されていること
- プラグインにより機能拡張できる安定したAPIを提供すること
- AI機能を無効化してもエディタ本体が完全に動作すること
- ソースコード・設計書・テストが長期保守可能な品質を満たすこと
